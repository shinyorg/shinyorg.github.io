2.6.0 August 9, 2022

Core

  • [Fix] InvokeTaskOnMainThread separated from InvokeOnMainThreadAsync to fix wrong signatures being used internally

BluetoothLE

  • [Enhancement][Android] Now only requests BLUETOOTH_SCAN and BLUETOOTH_CONNECT if necessary for API level 31 GH-1060

BluetoothLE Hosting

  • [BREAKING][Enhancement] All characteristic hooks are now async
  • [BREAKING][Android] All android specific properties and manufacturer data advertising options have been removed
  • [Enhancement][Android] IBleHostingManager.RequestAccess(bool advertise = true, bool connect = true) allows user to request appropriate permissions
  • [Enhancement][Android] Now only uses the appropriate on API level 31
  • [Enhancement][Android] LocalName is now advertised

Notifications

  • [BREAKING][Fix] Attachment (FileInfo) is now LocalAttachmentPath (string) to prevent confusion
  • [BREAKING] Notification.Sound is now an enum with the following values:
    • High
    • Default
    • Custom
    • None
  • [iOS][BREAKING][Enhancement] Time Sensitive notifications are no longer used by default as they require an extra entitlement being setup, thus you must now specify NotificationManager.UseTimeSensitiveAlerts = true; on iOS in order to enable them
  • [iOS][Fix] Time sensitive permission is only used if enabled, channel importance of high, and a scheduled date is set
  • [iOS][Enhancement] Local notification acknowledgement now uses Banner instead of Alert to remove iOS deprecation GH-1055
  • [Android][iOS][Fix] Notification.ImageUri is replaced with Notification.Attachment (FileInfo) as files must be local for attachments

HTTP Transfers

  • [iOS][Fix] Error messages for downloads are now provided GH-1048
